URL Encoding Reference for Database Passwords
================================================

When using special characters in your PostgreSQL password within a DATABASE_URL,
you need to URL-encode them. Here's a reference guide:

Special Characters and Their Encoded Values:
---------------------------------------------

Character    Encoded    Description
---------    -------    -----------
@            %40        At symbol
#            %23        Hash/Pound
%            %25        Percent
&            %26        Ampersand
+            %2B        Plus
=            %3D        Equals
?            %3F        Question mark
/            %2F        Forward slash
:            %3A        Colon
;            %3B        Semicolon
<            %3C        Less than
>            %3E        Greater than
[            %5B        Left bracket
]            %5D        Right bracket
{            %7B        Left brace
}            %7D        Right brace
|            %7C        Pipe
\            %5C        Backslash
^            %5E        Caret
~            %7E        Tilde
`            %60        Backtick
!            %21        Exclamation
$            %24        Dollar
'            %27        Single quote
"            %22        Double quote
(            %28        Left parenthesis
)            %29        Right parenthesis
*            %2A        Asterisk
,            %2C        Comma
-            %2D        Hyphen
.            %2E        Period
_            %5F        Underscore
(space)      %20        Space (or use +)

Examples:
---------

Password: my@password123
Encoded:  my%40password123
Full URL: postgresql://postgres:my%40password123@localhost:5432/leads_scrapping?schema=public

Password: P@ssw0rd#2024
Encoded:  P%40ssw0rd%232024
Full URL: postgresql://postgres:P%40ssw0rd%232024@localhost:5432/leads_scrapping?schema=public

Password: MyP@ss!2024
Encoded:  MyP%40ss%212024
Full URL: postgresql://postgres:MyP%40ss%212024@localhost:5432/leads_scrapping?schema=public

Password: Test@123#Pass
Encoded:  Test%40123%23Pass
Full URL: postgresql://postgres:Test%40123%23Pass@localhost:5432/leads_scrapping?schema=public

Quick Tips:
-----------
1. Only encode the password part, not the entire URL
2. The format is: postgresql://username:ENCODED_PASSWORD@host:port/database?schema=public
3. If your password has multiple special characters, encode each one
4. Common issue: @ symbol in password must be %40
5. You can use online URL encoders if needed: https://www.urlencoder.org/

How to Use:
-----------
1. Take your password
2. Replace each special character with its encoded value
3. Use the encoded password in your DATABASE_URL in .env file

Example Workflow:
-----------------
Original password:  Admin@2024#Secure
Step 1: Replace @ with %40  → Admin%402024#Secure
Step 2: Replace # with %23  → Admin%402024%23Secure
Final encoded password: Admin%402024%23Secure

DATABASE_URL format:
--------------------
postgresql://postgres:ENCODED_PASSWORD@localhost:5432/leads_scrapping?schema=public
